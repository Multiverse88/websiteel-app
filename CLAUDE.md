## What This Is

**EasyLegal** (`easylegal.id`) is an Indonesian business-legality services platform. It is a Next.js 16 App Router project (`websiteel-app/`) that markets and sells 12+ legal services (PT/CV establishment, HAKI trademarks, NIB/OSS, ISO certification, PKP tax, visa/KITAS, etc.), runs a content/blog system, and ships a JWT + TOTP-2FA admin dashboard. Public content is mostly Indonesian; admin UI is Indonesian with a `#990202` crimson theme.

The repository root also contains design/source materials extracted from Figma (`figma-*.png`, `*.txt`, `compact_*.txt`) and a `.qoder/repowiki/` autogenerated reference. Ignore these unless the user is specifically working on design parity or wiki content.

## Environment & Database

- `cp .env.example .env` for local dev. Required vars: `DATABASE_URL`, `DIRECT_URL`, `JWT_SECRET`. Optional: `SMTP_*`, `NEXT_PUBLIC_APP_URL`, `DISABLE_SECURE_COOKIES`.
- Database is PostgreSQL (use `host.docker.internal` from the app container to reach a host-installed Postgres, per `docker-compose.yml`).
- Prisma schema: `prisma/schema.prisma`. Migrations live in `prisma/migrations/`. Models: `User`, `Article`, `NewsletterSubscriber`, `NewsletterBroadcast`, `EmailLog`, `SystemSetting`. Author relation is optional with `onDelete: SetNull`.
- Seeding (`prisma/seed.ts`) creates the default admin (`admin@easylegal.id` / `admin123`, bcrypt cost 12), default system settings, and 3 starter articles.
- The Prisma client is the standard global-singleton pattern in `src/lib/db.ts` (dev hot-reload safe).

## High-Level Architecture

**Hybrid rendering, Server-Action backend.** There are no `app/api/*` route handlers — every mutation goes through Server Actions (`"use server"` files like `src/app/login/actions.ts`, `src/app/dashboard/newsletter/actions.ts`, `src/app/dashboard/artikel/tambah/actions.ts`).

```
websiteel-app/src
├── app/                # App Router pages + Server Actions
│   ├── (public)        # /, /tentang-kami, /kontak, /artikel, /artikel/[slug],
│   │                   # /cek-nama, /cek-kbli, /referral-reseller, /kerjasama,
│   │                   # /layanan/*, /home-gads
│   ├── login/          # /login + 2FA setup/verify Server Actions
│   └── dashboard/      # Protected (see proxy.ts); article CMS, newsletter, profile
├── components/         # Shared (Navbar, Footer, CTA, FAQ, Pricing, …)
│   ├── home/           # Homepage section components (Hero, LayananKami, etc.)
│   ├── layanan/        # Shared service-page templates (e.g. BadanUsahaTemplate)
│   └── ui/             # shadcn primitives (accordion, button, …)
├── data/               # Static content data (e.g. layanan-badan-usaha.tsx)
├── lib/                # auth.ts, 2fa.ts, db.ts, mail.ts, utils.ts
└── proxy.ts            # Next.js middleware → protects /dashboard/* via JWT
```

### Authentication flow
1. `POST /login` → `loginAction` (Server Action) bcrypt-verifies against `User`, sets a 5-min `pending_2fa_user` cookie, redirects to `/login/setup-2fa` (first time) or `/login/verify-2fa` (returning).
2. 2FA actions live in `src/app/login/actions-2fa.ts` and use helpers in `src/lib/2fa.ts` (`otplib` + `qrcode`).
3. On success, `createSession` (`src/lib/auth.ts`) mints a 24h HS256 JWT (`jose`) into the `admin_token` httpOnly cookie.
4. `src/proxy.ts` is the Next.js middleware (matcher: `/dashboard`, `/dashboard/:path*`); on every request it calls `jwtVerify` and redirects to `/login` if invalid.
5. Each dashboard Server Component also calls `getSession()` defensively and re-checks `twoFactorEnabled` (see `src/app/dashboard/page.tsx`).

### Service-page pattern
`/layanan/<slug>` pages follow one of two patterns:
- **Template-driven**: pages that share a body template (e.g. `pendirian-badan-usaha/page.tsx` → `BadanUsahaTemplate` reading from `src/data/layanan-badan-usaha.tsx`, with sub-routes under `[jenis]/page.tsx`).
- **Self-contained**: long single-file pages (e.g. `merek-haki/page.tsx`, `nib-oss/page.tsx`, `press-release/page.tsx`) that import the shared `Navbar`/`Footer` via the root layout and compose their own sections.

There is a permanent redirect from `/layanan/pendirian-yayasan` → `/layanan/pendirian-badan-usaha/yayasan` (declared in `next.config.ts`).

### Article system
- **Listing** (`src/app/artikel/page.tsx`): SSR, `dynamic = "force-dynamic"`, auto-seeds 3 default articles if DB is empty.
- **Detail** (`src/app/artikel/[slug]/page.tsx`): SSR, custom markdown renderer in-file (supports `### headings`, `**bold**`, bullet/numbered lists, `---`); auto-generated ToC (`table-of-contents.tsx`); client-side `view-tracker.tsx` increments `viewCount`.
- **CMS** (`src/app/dashboard/artikel/tambah/`): client-component with live preview, file upload (5MB limit) or URL mode, 4 Unsplash preset images, calls `createArticle` Server Action.

### Newsletter system
- Subscribe via the widget on article pages → `NewsletterSubscriber` row.
- Admin broadcasts from `/dashboard/newsletter` → loops active subscribers, sends via Nodemailer, logs each attempt to `EmailLog` linked to a `NewsletterBroadcast` parent.
- If `SMTP_*` env vars are absent, `src/lib/mail.ts` falls back to a console simulation (`status: "simulated"`).
- `SystemSetting` key/value store drives auto-broadcast on new article creation (see dashboard `actions.ts`).

### Animations
- GSAP is the primary animation library; `ScrollManager.tsx` (mounted in root layout) handles smooth-scroll for same-page links + hash navigation on route change using `ScrollToPlugin`.
- `template.tsx` is currently a thin wrapper. Page transitions are largely handled by per-component `useGSAP` hooks (e.g. on the homepage).
- Framer Motion is used for some `whileInView` reveals. Do not introduce a third animation library.

## Docker / Deployment

Multi-stage `Dockerfile` (deps → builder → runner, all `node:20-alpine`, `apk add openssl` for Prisma). Final image runs `npx prisma@6.19.3 migrate deploy && node server.js`, so migrations apply on container start. `next.config.ts` sets `output: "standalone"` for small image size.

`docker-compose.yml` runs two services:
- `app` — Next.js container, reads `.env.production`, mounts the `uploads` Docker volume at `/app/public/uploads`.
- `nginx` — `nginx:alpine` reverse proxy, exposes 80/443, mounts host Let's Encrypt certs, shares the `uploads` volume read-only.

PostgreSQL is expected to be installed directly on the host VPS (not in Compose). Inside the app container, `host.docker.internal:host-gateway` resolves to the host — already configured via `extra_hosts` in Compose.

VPS scripts in `websiteel-app/scripts/`:
- `deploy-setup.sh` — one-time VPS bootstrap (Docker, certbot, UFW, backup dir).
- `deploy.sh` — `git pull && docker compose build --no-cache && docker compose up -d`.
- `backup-db.sh` — `pg_dump` to `/opt/backups/easylegal_*.sql.gz`, auto-cleans >7 days.
- `secure-vps.sh` — hardening helper.

Nginx config in `websiteel-app/nginx/nginx.conf` is wired to `easylegal.my.id` (note: the `DEPLOYMENT.md` and `.env.example` mention `easylegal.id` — be aware both names appear in different places; the live cert paths follow the nginx file).

## Design / Styling

- Tailwind v4 via `@tailwindcss/postcss` (no `tailwind.config.js`; tokens declared inside `src/app/globals.css` `@theme { … }`).
- Custom color tokens: `primary #D62828`, `primary-light #FEF2F2`, `primary-hover #B91C1C`, admin `crimson #990202`, `whatsapp #25D366`. Custom breakpoint `xs: 375px`.
- Fonts loaded via `next/font/google` in `src/app/layout.tsx`: DM Sans (body), Plus Jakarta Sans (secondary), Inter (display/headings).
- Reusable CSS classes defined in `globals.css`: `.glassmorphism`, `.custom-shadow`, `.custom-shadow-lg`, plus `animate-*` keyframes (`slideDown`, `fadeInUp`, `float`, `pulse-subtle`, `fadeIn`, `marquee-left/right`).
- shadcn/ui (style `new-york`, base `neutral`) is installed. Add components with `npx shadcn@latest add <name>`. Custom registries include `@react-bits`.

## Conventions to Follow

- Path alias `@/*` → `src/*` (see `tsconfig.json`).
- All new mutation endpoints → Server Actions with `"use server"`, not API routes.
- Dashboard pages start with `export const dynamic = "force-dynamic";` and a defensive `getSession()` + `redirect("/login")` check.
- For long static service pages, prefer a single self-contained `page.tsx` over fragmenting into many components; the existing service pages (700–1500 lines each) are the reference style.
- Public lead-gen flows (kontak, cek-nama, cek-kbli, referral, kerjasama) all funnel to WhatsApp (`https://wa.me/6281123456789?text=...`) rather than persisting to the DB.
- Generated image folders under `public/uploads/` are gitignored (only `.gitkeep` placeholders are kept). Do not commit uploaded assets.
- `.env.production`, `nginx/certs/`, and `*.pem` must never be committed.
